# 测试策略

## 文档阶段

- 检查必须文件是否存在。
- 检查数据库脚本是否包含六张 MVP 表。
- 检查 API 文档是否覆盖客户池、AI、Landing 和 Analytics。

## 后端阶段

- Unit Test：实体规则、状态流转、Service 逻辑。
- Integration Test：API、数据库、权限和多租户隔离。
- Contract Test：OpenAPI 与前端 API Client 一致性。

## 前端阶段

- Component Test：表单校验、列表筛选、状态标签。
- E2E Test：客户提交 Landing → 客户池出现 → 销售修改状态 → 数据分析更新。

## AI 阶段

- Prompt Snapshot：同一输入下输出结构稳定。
- Safety Test：过滤夸大承诺、违规词和敏感内容。
