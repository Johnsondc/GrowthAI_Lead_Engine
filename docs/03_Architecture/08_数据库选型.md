# 数据库选型

## 结论

V1 使用 MySQL 8。

## 原因

- MVP 面向中小企业和本地生活 SaaS，MySQL 部署成本低、团队熟悉度高。
- MySQL 8 支持 JSON 字段，可保存 AI 任务输入。
- Docker 本地开发简单，云厂商托管选择多。
- 当前业务主要是结构化客户线索、来源、内容和统计，MySQL 足够支撑第一阶段。

## 什么时候重新评估

- 需要复杂 BI、列式分析或超大规模报表。
- 进入强微软生态客户，且对 SQL Server 有明确要求。
- 需要更强全文检索或向量检索，可以单独引入 OpenSearch、Meilisearch 或向量数据库，而不是替换主库。
